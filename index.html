How the C compiler is written by C

2019-09-25

Read this on an article about self-reproducing c programs:

The C compiler is written in C.
– Reflections on Trusting Trust

It sounds like a chicken-and-egg paradox. But actually this is simple: you write the most basic parts of C by Assembly, then you use them to build the rest of C. This is known as bootstrapping (wiki)). As they will all envetually be interpreted as Assembly, it makes no difference to the computer (but make human’s work much easier).

The Golang actually does the same. The basic of Golang is written by C and compiled by gcc. Then they use the basic Golang to implement the Golang compiler (gccgo). You can find gccgo information on the Golang github page here.

tf-Faster-RCNN安装排坑

2019-05-21

overview

github地址

GTX970对应的架构是sm_52。

  • 系统： Ubuntu16.04
  • 显卡： GTX970
  • cuda版本： 9.0 + 对应版本cudnn（7.5.1）
  • tensorflow版本：1.12.0 （对应cuda，1.13只支持cuda10.0）

按照github上面的就都ok，缺什么module就安什么。如果安好之后，没有使用GPU，就是依赖没安装对。实际上就是tensorflow+cuda+cudnn没安装对。

安装tensorflow

要同时安装tensorflow和tensorflow-gpu，使用pip install就可以，但是因为cuda9.0只能和tensorflow r1.12.0使用，所以要指定tf版本。

sudo pip install tensorflow==1.12.0 -i <国内源>
sudo pip3 install tensorflow-gpu==1.12.0 -i <国内源>
sudo pip install tensorflow==1.12.0 -i <国内源>
sudo pip3 install tensorflow-gpu==1.12.0 -i <国内源>

安装好使用

pip list | grep tensorflow

查看，注意tensorflow和tensorflow gpu版要对应，tf-estimator也要对应。不行就全都卸载了重新安装。

验证方式也简单，

不报错就好了。

安装cuda

之后安装cuda，首先从ubunut设置里面安装好驱动，384.xx那个。

cuda安装去官网找就好，使用runfile方式安装更简单，注意已经有驱动了，不需要安装驱动。

cuda安好之后记得设置path。

cuda安装好了记得验证，官网有写怎么验证。

安装cudnn

cudnn下载界面特别懒没任何介绍，他们实际上是有一个doc的。会告诉你下载四个文件，其中三个是lib。

安装好之后还会有个example让你测试，会报错INTERNAL什么的，百度一下会让你rm什么玩意，rm了之后make clean && make就好了。

最坑的是cuda，cudnn，tensorflow这三个都要好了才能用GPU，不然就会用CPU，跑的很慢。
之前我跑要3.5s*5000的时间，200多分钟，安好了之后用970跑是20分钟。

    <div class="container">

    </div>
</footer>

之后在原本需要放置页脚代码的地方插入：

<script>
        $(function(){
            $("#footer").load("footer.html");
        });
    </script>
