tf-Faster-RCNN安装排坑

2019-05-21

overview

github地址

GTX970对应的架构是sm_52。

  • 系统： Ubuntu16.04
  • 显卡： GTX970
  • cuda版本： 9.0 + 对应版本cudnn（7.5.1）
  • tensorflow版本：1.12.0 （对应cuda，1.13只支持cuda10.0）

按照github上面的就都ok，缺什么module就安什么。如果安好之后，没有使用GPU，就是依赖没安装对。实际上就是tensorflow+cuda+cudnn没安装对。

安装tensorflow

要同时安装tensorflow和tensorflow-gpu，使用pip install就可以，但是因为cuda9.0只能和tensorflow r1.12.0使用，所以要指定tf版本。

sudo pip install tensorflow==1.12.0 -i <国内源>
sudo pip3 install tensorflow-gpu==1.12.0 -i <国内源>
sudo pip install tensorflow==1.12.0 -i <国内源>
sudo pip3 install tensorflow-gpu==1.12.0 -i <国内源>

安装好使用

pip list | grep tensorflow

查看，注意tensorflow和tensorflow gpu版要对应，tf-estimator也要对应。不行就全都卸载了重新安装。

验证方式也简单，

不报错就好了。

安装cuda

之后安装cuda，首先从ubunut设置里面安装好驱动，384.xx那个。

cuda安装去官网找就好，使用runfile方式安装更简单，注意已经有驱动了，不需要安装驱动。

cuda安好之后记得设置path。

cuda安装好了记得验证，官网有写怎么验证。

安装cudnn

cudnn下载界面特别懒没任何介绍，他们实际上是有一个doc的。会告诉你下载四个文件，其中三个是lib。

安装好之后还会有个example让你测试，会报错INTERNAL什么的，百度一下会让你rm什么玩意，rm了之后make clean && make就好了。

最坑的是cuda，cudnn，tensorflow这三个都要好了才能用GPU，不然就会用CPU，跑的很慢。
之前我跑要3.5s*5000的时间，200多分钟，安好了之后用970跑是20分钟。

以添加所有的页脚为例：

新建一个footer.html，在里面输入

<footer class="footer mt-auto py-3">
    <div class="container">
        <span class="text-muted">联系开发者: tolkkki@outlook.com</span>
    </div>
</footer>

之后在原本需要放置页脚代码的地方插入：

<script>
        $(function(){

        });
    </script>
    <div id="footer"></div>

设置hexo的favicon

2019-04-20

1.生成图标文件

选好想要的图片，用photoshop裁剪和修改image_size，调整为32x32像素的PNG格式（不是PNG的话背景不能透明）。

谷歌一个在线生成ico格式的网站，转为.ico格式。

2.放到博客里

将图片放到路径：themes/主题名/source/image/favicon.ico （没有image文件夹自己建一个）。

将thems/主题名/_config.yml 中的favicon字段设置为图片的（相对）路径：

# _config.yml

favicon: /image/favicon.ico

完成。

使用crontab

2018-11-05

crontab（好像有的linux发行版叫cron）的作用是规定的时间内执行某个命令。
当初学这个是为了每天整点执行签到脚本。

添加任务

crontab -e

时间格式：（每个整点执行myshell.sh）

0 * * * * . /etc/profile; /bin/bash /home/myshell.sh

注意必须指定sh路径

脚本文件中必须使用绝对路径，否则会使用的路径是~/

查看任务

crontab -l

不放心就重启一下服务

sudo service cron restart

公钥加密的应用

2018-08-01

因为我是信息安全专业的，面试的时候就经常被问到这个，很神奇……

实际上光RSA的应用就太多了，说2个最最常见的吧

1. TLS/SSL

TLS加密首先用公钥加密协商一个私钥出来。

之后用这个协商出来的私钥进行对称加密传输。

（以下摘自维基百科：）

为了生成会话密钥用于安全连接，客户端使用服务器的公钥加密随机生成的密钥，并将其发送到服务器，只有服务器才能使用自己的私钥解密。

利用随机数，双方生成用于加密和解密的对称密钥。这就是TLS协议的握手，握手完毕后的连接是安全的，直到连接（被）关闭。如果上述任何一个步骤失败，TLS握手过程就会失败，并且断开所有的连接。

这样做的好处：

  1. 因为公钥加密效率太低，实际传输时必须用对称加密。
  2. 对称加密的密钥每次都会更换，由于公钥加密的特点，可以通过加密传递私钥。

2. SSH的RSA签名登录

这个在git中有非常广泛的应用，下面介绍一下原理：

假设你有一台自己的PC，叫做A；一台服务器，叫做B。

（*事实上，在我们使用github的时候，我们自己的电脑就是A，github就是B）

你尝试在A上通过SSH登录B。但是每次都要输入SSH密钥，很麻烦，起的太简单又总有弱智破你的弱密钥来DDos，怎么办？

简单，你可以在A上生成一对RSA公钥和密钥，分别存在两个文件里。

然后，把公钥文件拷贝到B上，设置B成为能提供这个公钥的对应私钥的人就可以登录。

（这个过程在github上也是手动的，github的settings中SSH选项里一般都有你之前设置的RSA，只不过你可能已经忘了）

注意这里不是使用RSA加密信息，而是使用RSA签名来认证。
